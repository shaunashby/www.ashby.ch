System Administration


The ISDC, a growing astronomical data centre, featured


  1. • 50 on-site users

  2. • 75+ off-site users

  3. • 130 servers totalling 200U of rack space

  4. • 9 diverse projects (INTEGRAL, Gaia, CTA, Polar, Astro-H, Planck, AHEAD, HEAVENS, FACT)

  5. • 3 buildings

  6. • 4 machine rooms


ISDC system management team responsibilities included


  1. - Operating systems (Linux, Mac OS X, Solaris 8, FreeBSD)

  2. - Remote access (SSH, NoMachine, Microsoft Terminal Services)

  3. - Network cabling, switches/routers and basic infrastructure (DHCP, DNS, NTP)

  4. - Authentication and authorization (NIS, LDAP)

  5. - Provisioning, configuration and monitoring (Kickstart, Puppet, Nagios, Ganglia)

  6. - Network attached storage (NFS, Samba, sshfs)

  7. - Web servers (Apache, MediaWiki, mod_perl2, Tomcat, legacy CGI scripts)

  8. - Data distribution (FTP, Rsync)

  9. - Database servers (MySQL, PostgreSQL)

  10. - Software development support (Trac, Agilo, Bitten, Subversion, Git)

  11. - Printers, scanners and printing infrastructure (CUPS)

  12. - On-call 24x7 support for critical INTEGRAL operations (3 hour on-site response), one week in three

  13. - Support legacy Solaris servers and storage (Sun V280R, Apple Xserve RAID)

  14. - Two compute clusters totalling 40 nodes (Rocks Linux)

  15. - User servers with many scientific/astronomical software packages

  16. - Computing infrastructure for visitors

  17. - Specification, sourcing and purchase of computer equipment for each individual user

  18. - Specification, sourcing, installation and configuration of new server equipment for all projects

  19. - Backups of system machines (Bacula, legacy shell scripts)

  20. - Backups of user machines (BackupPC, AppleTalk, TimeMachine)

  21. - Renewal and replacement of legacy systems

  22. - Software development support for software team

  23. - Liaison with University computing services (email, network infrastructure)

  24. - Running monthly meetings to inform management and set direction

  25. - Tracking emerging technology trends (e.g. SSDs, virtualisation, "NoSQL") and advising the software team
